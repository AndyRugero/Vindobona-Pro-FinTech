# Lesson 44: Two-Factor Authentication (2FA) with Google Authenticator 📱🔐🔑

Welcome to Lesson 44! Now that we have secured our authentication endpoints against brute-force attacks with rate limiters, we are ready to implement the gold standard of modern web security: **Two-Factor Authentication (2FA)**.

Two-Factor Authentication adds an extra layer of defense by requiring something the user **knows** (their password) AND something the user **has** (their physical mobile phone generating rolling security codes).

---

## 1. What is 2FA and TOTP? 🎟️📱

In this lesson, we will use **TOTP (Time-Based One-Time Passwords)**. The server and the user's mobile app (such as Google Authenticator, Microsoft Authenticator, or Authy) share a secret cryptographic key. Both devices use the current time to compute a matching 6-digit code that changes every 30 seconds.

### 🗺️ The Analogy:
Think of 2FA like a **Bank Safe Deposit Box** 🏦🔐:
*   **The First Lock:** Requires the customer's key (their password).
*   **The Second Lock:** Requires the bank manager's guard key (the rolling 6-digit code generated by the Authenticator app on their phone).
*   Even if a hacker steals your password, they cannot open the safe because they don't hold your physical phone to turn the second lock!

## 2. How the 2FA Flow Works 🔄

Here is the handshake flow for enabling and logging in with 2FA:

### Phase A: Enabling 2FA (Setup)
```text
[ React Frontend ]                         [ Express Backend ]                   [ SQLite Database ]
        |                                           |                                     |
        | -- 1. Request 2FA Setup -------------->   | -- 2. Generate secret string -----> | (Temporarily store
        |                                           |       (e.g., KVKFK...)              |  secret for user)
        |                                           |                                     |
        | <-- 3. Return QR Code URL & Secret -------|                                     |
        |                                           |                                     |
        | -- 4. Scan QR code with Phone App         |                                     |
        |       (App starts generating codes)       |                                     |
        |                                           |                                     |
        | -- 5. Submit 1st 6-digit code to verify ->| -- 6. Verify code matches time ---> | (Set 2fa_enabled = 1
        |                                           |       and save permanent secret     |  in users table)
```

### Phase B: Logging In with 2FA
```text
[ React Frontend ]                         [ Express Backend ]                   [ SQLite Database ]
        |                                           |                                     |
        | -- 1. Login (username/password) --------> | -- 2. Validate password ------------|
        |                                           |       Check if 2FA is active        |
        |                                           |                                     |
        | <-- 3. Respond: "2FA Code Required" ------|                                     |
        |                                           |                                     |
        | -- 4. Prompt user for 6-digit phone code   |                                     |
        |                                           |                                     |
        | -- 5. Submit 6-digit code --------------> | -- 6. Verify code matches secret -> | (Read user's secret)
        | <-- 7. Success: Return JWT Token ---------|                                     |
```

## 3. The Rosetta Stone: C# vs. Node.js 🔍

If you have built secure portals in ASP.NET Core, you have used standard MFA packages:

| ASP.NET Core (C#) | Node.js (Express) | Purpose |
| :--- | :--- | :--- |
| `Microsoft.AspNetCore.Identity` / TOTP Helpers | `otplib` | Cryptographic engine that generates and verifies TOTP codes. |
| `QRCoder` | `qrcode` | Package that converts the TOTP secret URL into a scannable QR Code image. |
| `TwoFactorEnabled` | `two_factor_enabled` (DB Column) | Database boolean flag indicating if the user has completed setup. |
